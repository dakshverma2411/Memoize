# Security Policy

## Supported Versions

The following versions of Memoize are currently supported with security updates:

| Version | Supported          |
| ------- | ------------------ |
| 0.x     | :white_check_mark: |

## Reporting a Vulnerability

**Please do NOT open public GitHub issues for security vulnerabilities.**

If you discover a security vulnerability in Memoize, we appreciate your help in disclosing it responsibly.

### How to Report

You have two options for reporting security vulnerabilities:

1. **GitHub Security Advisories (Preferred)**: Use [GitHub's Security Advisory feature](https://github.com/dakshverma2411/memoize/security/advisories/new) to privately report the vulnerability.

2. **Email**: Send an email to the project maintainers with details of the vulnerability.

### What to Include

When reporting a vulnerability, please include:

- A description of the vulnerability
- Steps to reproduce the issue
- The potential impact of the vulnerability
- Any possible mitigations you have identified
- Your name/handle for attribution (optional)

## Response Timeline

We take all security reports seriously. Here is what you can expect:

| Action                          | Timeline          |
| ------------------------------- | ----------------- |
| Acknowledgment of report        | Within 48 hours   |
| Initial assessment              | Within 1 week     |
| Resolution target (critical)    | Within 2 weeks    |
| Resolution target (non-critical)| Within 30 days    |
| Public disclosure               | After fix release  |

We will keep you informed of our progress throughout the process. If for some reason we are unable to meet these timelines, we will notify you with an updated timeline.

## Disclosure Policy

- We will coordinate disclosure with reporters
- We will credit reporters in security advisories (unless anonymity is requested)
- We will publish security advisories through GitHub once a fix is available
